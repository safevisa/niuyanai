# 微信支付回调联调说明（当前版本）

本文档对应当前后端实现，目标是让你在部署环境快速完成“下单 -> 回调 -> 升级会员”的联调。

## 1. 环境变量

在后端环境变量中配置：

- `WECHAT_PAY_CALLBACK_SECRET`：微信回调签名密钥（服务端与回调发送方一致）

未配置时，`/api/payments/wechat/callback` 会返回 `501`。

## 2. 现有支付接口

### 2.1 创建订单

- `POST /api/payments/create`
- 需要登录（`Authorization: Bearer <token>`）
- 请求体：

```json
{
  "plan": "vip"
}
```

可选值：`vip` / `pro`

返回示例：

```json
{
  "status": "success",
  "data": {
    "order_no": "BY20260505192000123456",
    "plan": "vip",
    "amount": 39.0,
    "payment_url": "weixin://mockpay?order_no=BY20260505192000123456",
    "status": "pending"
  }
}
```

### 2.2 查询订单列表

- `GET /api/payments/orders`
- 需要登录

### 2.3 查询单个订单（前端轮询用）

- `GET /api/payments/orders/{order_no}`
- 需要登录

### 2.4 Mock确认支付（开发联调用）

- `POST /api/payments/mock-confirm`
- 需要登录
- 请求体：

```json
{
  "orderNo": "BY20260505192000123456"
}
```

## 3. 真实微信回调接口

### 3.1 接口信息

- `POST /api/payments/wechat/callback`
- Header: `X-Wechat-Signature: <hex_sha256_hmac>`
- Body（JSON）：

```json
{
  "orderNo": "BY20260505192000123456",
  "status": "paid"
}
```

说明：

- `orderNo` 必填，对应支付订单号
- `status`：
  - `paid`：支付成功，会升级用户会员
  - 其他值：标记订单 `failed`

### 3.2 签名算法

签名串 = 请求原始 body（字节流）

签名值 = `HMAC_SHA256(secret=WECHAT_PAY_CALLBACK_SECRET, msg=raw_body)`，输出小写十六进制字符串

后端会将 Header 中的 `X-Wechat-Signature` 与计算值进行常量时间比较。

## 4. 签名示例

### 4.1 Python

```python
import hmac
import hashlib
import json

secret = "your_secret_here"
payload = {
    "orderNo": "BY20260505192000123456",
    "status": "paid",
}
raw = json.dumps(payload, separators=(",", ":"), ensure_ascii=False).encode("utf-8")
sig = hmac.new(secret.encode("utf-8"), raw, hashlib.sha256).hexdigest()
print(sig)
```

### 4.2 Node.js

```js
const crypto = require("crypto");

const secret = "your_secret_here";
const payload = {
  orderNo: "BY20260505192000123456",
  status: "paid",
};
const raw = JSON.stringify(payload);
const sig = crypto.createHmac("sha256", secret).update(raw).digest("hex");
console.log(sig);
```

## 5. curl 联调示例

```bash
curl -X POST "https://your-domain/api/payments/wechat/callback" \
  -H "Content-Type: application/json" \
  -H "X-Wechat-Signature: <你的签名>" \
  -d '{"orderNo":"BY20260505192000123456","status":"paid"}'
```

## 6. 常见错误码

- `501`: 未配置 `WECHAT_PAY_CALLBACK_SECRET`
- `401`: 签名校验失败
- `400`: 缺少 `orderNo`
- `404`: 订单不存在 / 订单用户不存在

## 7. 当前状态机（简化）

- 下单：`pending`
- 成功回调：`paid`（写入订阅并升级会员）
- 失败回调：`failed`

> 备注：当前版本是可联调骨架，真实生产接入微信官方参数时，只需把回调字段映射到 `orderNo/status` 并按同样方式计算验签即可。
